# 简历展示准备

## 需要准备的截图

为了在简历和 GitHub README 中展示项目效果，需要准备以下截图：

### 1. **总览截图** (`screenshot-overview.png`)
- **内容**：完整的四层价值栈 + 时间轴视图
- **建议**：使用"AI生态系统示例"，展示完整布局
- **要点**：
  - 显示所有四个层级
  - 显示多个发展阶段
  - 显示原爆点和不动点
  - 使用深色主题（更专业）

### 2. **层级细节截图** (`screenshot-layers.png`)
- **内容**：突出价值栈的层级结构
- **建议**：聚焦到某一个层级，展示公司卡片的详细信息
- **要点**：
  - 显示公司属性标签
  - 显示渐变色效果
  - 显示连线关系

### 3. **侧边栏功能截图** (`screenshot-sidebar.png`)
- **内容**：侧边栏的原爆点设置、不动点管理、公司列表
- **要点**：
  - 展示工具的交互界面
  - 展示配置功能

### 4. **交互演示 GIF** (`demo-interaction.gif`)
- **内容**：拖拽、连线、右键菜单的操作演示
- **时长**：10-15 秒
- **建议使用工具**：
  - macOS: QuickTime Player（录屏） + Gifski（转 GIF）
  - 在线工具: https://ezgif.com/

### 5. **简历用精简截图** (`resume-screenshot.png`)
- **内容**：最能体现项目价值的单张截图
- **建议**：总览截图，突出四层价值栈和时间轴
- **尺寸**：适合打印和屏幕查看（建议 1920x1080）

---

## 截图制作指南

### 使用本地 HTML 制作截图

1. 用浏览器打开 `ecosystem-evolution-demo.html`
2. 点击"导入数据"，加载"AI生态系统示例.json"（最完整）
3. 调整画布视图：
   - 使用深色主题
   - 调整缩放比例，确保内容完整可见
   - 隐藏不必要的元素
4. 截图工具：
   - macOS: `Cmd + Shift + 4`
   - Windows: `Win + Shift + S`

### 截图优化建议

- **分辨率**：至少 1920x1080
- **格式**：PNG（保留清晰度）
- **压缩**：使用 TinyPNG 等工具压缩（减小文件体积）
- **标注**：如需标注，使用简洁的箭头和文字

## 简历中如何使用

### 方案 1：附件形式
- 将精简截图作为简历附件
- 文件命名：`生态可视化项目-项目截图.png`

### 方案 2：嵌入简历
- 在项目描述下方插入小图
- 尺寸控制在 600x400 左右
- 确保打印时清晰可见

### 方案 3：链接形式
- 简历中只放 GitHub 链接
- GitHub README 中嵌入完整截图
