# 使用指南：从零开始分析一个新生态

本指南将带你从零开始，用生态演化框架分析一个行业。

---

## 准备工作

### 工具准备
- 下载并打开本地 `ecosystem-evolution-demo.html` 文件
- 准备行业基础研究资料

### 时间预算
- 初步框架：1-2 小时
- 深度分析：1-2 天（持续迭代）

---

## 案例：分析"商业航天"生态

让我们以"商业航天"为例，完整演示分析流程。

---

## 第一步：定义原爆点

### 1.1 找到核心突破

**问自己三个问题**：
1. 这个行业是什么时候"爆发"的？
2. 背后的技术/模式突破是什么？
3. 消除了什么关键约束？

**商业航天案例**：

| 问题       | 答案                                                                 |
| ---------- | -------------------------------------------------------------------- |
| 爆发时间   | 2015 年左右（SpaceX 首次回收火箭）                                   |
| 技术突破   | **可回收火箭**（Falcon 9）                                           |
| 消除的约束 | **发射成本**（从 5000万美元/次 → 6000万美元/次，但边际成本大幅降低） |

**原爆点定义**：  
```
SpaceX 可回收火箭技术，将发射成本降低 10 倍，
消除了商业航天的经济性约束。
```

### 1.2 在工具中操作

1. 点击顶部"原爆点"标签
2. 填写：
   - **名称**："可回收火箭技术"
   - **描述**："SpaceX Falcon 9 首次回收成功（2015），将发射成本降低 10 倍，消除商业航天的经济性约束"

---

## 第二步：识别不动点

### 2.1 找到"不变的结构"

**从以下角度思考**：
- 竞争格局（双寡头？三足鼎立？）
- 循环结构（飞轮效应？网络效应？）
- 技术分叉（开源 vs 闭源？国有 vs 民营？）
- 价值流动（中间件？API 经济？）

**商业航天案例**：

| 不动点                 | 符号 | 说明                                                 |
| ---------------------- | ---- | ---------------------------------------------------- |
| **国有 vs 民营分叉**   | ×    | SpaceX（民营） vs NASA/波音/洛马（国有体系）         |
| **发射-卫星-应用闭环** | □    | 火箭发射 → 卫星组网 → 数据应用，形成平台生态         |
| **数据飞轮**           | ○    | 更多卫星 → 更多数据 → 更多应用 → 更多收入 → 更多卫星 |

### 2.2 在工具中操作

1. 点击"不动点"标签
2. 分别点击符号按钮（×、□、○）添加不动点：
   - × "国有 vs 民营分叉"
   - □ "发射-卫星-应用平台"
   - ○ "数据飞轮"

---

## 第三步：拆解价值栈

### 3.1 四层分类

将公司/要素分配到四个层级：

**要素层**：
- SpaceX（火箭技术）
- Blue Origin（火箭技术）
- 卫星制造商（Planet Labs、OneWeb）

**基础设施层**：
- AWS Ground Station（地面站服务）
- Azure Space（云服务+卫星数据）
- Starlink（卫星互联网基础设施）

**场景入口层**：
- Planet Labs（遥感数据应用）
- Spire（海事/天气数据）
- Swarm Technologies（IoT 连接）

**规则标准层**：
- FCC（频谱分配）
- ITU（国际标准）
- Space Force（军事采购标准）

### 3.2 在工具中操作

1. 点击"公司/载体"标签
2. 点击"添加公司"
3. 填写信息：
   - **名称**："SpaceX"
   - **类型**："要素层"
   - **阶段**："成长期"（稍后定义）
   - **属性标签**：火箭、可回收、民营
   - **颜色**：选择渐变色

**重复添加其他公司**，逐步构建完整的价值栈。

---

## 第四步：划分发展阶段

### 4.1 时间轴设计

**商业航天案例**：

| 阶段       | 时间      | 标志性事件                      | 主要玩家                      |
| ---------- | --------- | ------------------------------- | ----------------------------- |
| **萌芽期** | 2002-2014 | SpaceX 成立，多次试验失败       | SpaceX、Blue Origin           |
| **探索期** | 2015-2019 | Falcon 9 首次回收，商业模式验证 | SpaceX、OneWeb、Planet Labs   |
| **成长期** | 2020-2024 | Starlink 商用，行业爆发         | SpaceX、AWS、Azure Space      |
| **扩张期** | 2025+     | 月球/火星任务，军事应用         | SpaceX、Blue Origin、中国航天 |

### 4.2 在工具中操作

1. 点击"配置"标签
2. 在"发展阶段"部分，修改默认阶段：
   - 萌芽期（2002-2014）
   - 探索期（2015-2019）
   - 成长期（2020-2024）
   - 扩张期（2025+）

---

## 第五步：建立连线关系

### 5.1 标注关键关系

**商业航天案例**：

- SpaceX → Starlink（内部垂直整合）
- Starlink → AWS Ground Station（基础设施合作）
- Planet Labs → Google Cloud（数据托管）
- SpaceX → NASA（政府采购）

### 5.2 在工具中操作

1. 在画布上，**拖拽公司位置**，让它们处于对应的层级和阶段
2. **右键点击**公司卡片，选择"添加连线"
3. 点击目标公司，完成连线

---

## 第六步：迭代和验证

### 6.1 自我检查清单

- [ ] 原爆点是否消除了关键约束？
- [ ] 不动点是否在演化中始终存在？
- [ ] 价值栈的层级划分是否合理？
- [ ] 发展阶段的时间节点是否准确？
- [ ] 连线关系是否反映了价值流动？

### 6.2 团队讨论

1. **导出 JSON 数据**：点击"导出数据"按钮
2. **分享给团队**：发送 JSON 文件或在线链接
3. **收集反馈**：询问"哪里不合理？哪里需要补充？"
4. **增量迭代**：用 AI 生成补充的 JSON，导入工具

---

## 第七步：输出与应用

### 7.1 截图与报告

1. 调整画布视图，突出关键部分
2. 截图保存
3. 插入到研报/PPT 中

### 7.2 持续更新

- **定期回顾**：每季度更新一次
- **新事件标注**：行业发生重大事件时，及时更新连线和阶段
- **版本管理**：保存不同时间点的 JSON 文件，可回顾演化过程

## 常见问题 FAQ

### Q1：我该从哪个步骤开始？

**A**：按顺序最好，但如果你已经有行业理解，可以：
1. 先画公司分布（价值栈 + 阶段）
2. 再倒推原爆点和不动点
3. 最后补充连线关系

### Q2：如果找不到明确的原爆点怎么办？

**A**：可能这个行业还没有真正的"生态演化"，或者是渐进式创新。例如：
- ❌ 不适合：消费品行业（没有技术奇点）
- ✅ 适合：新兴技术生态（AI、Crypto、商业航天）

### Q3：价值栈的层级划分很模糊怎么办？

**A**：遵循原则：
- **要素层**：提供核心技术/硬件/算法
- **基础设施层**：提供工具/平台/服务
- **场景入口层**：直接面向用户/客户
- **规则标准层**：制定标准/监管/开源协议

如果一家公司跨多个层级，可以标注多个卡片。

### Q4：如何避免画得太复杂？

**A**：聚焦核心：
- ✅ 只画头部公司（市值 Top 10）
- ✅ 只画关键连线（主要合作/竞争关系）
- ❌ 不要试图覆盖所有公司

**记住**：80%的价值在 20%的要素上。

## 进阶技巧

### 技巧 1：用 AI 生成 JSON

1. 将你的行业理解整理成文字
2. 让 AI（如 ChatGPT）按照 [JSON 格式说明](JSON-FORMAT.md) 生成数据
3. 导入工具，快速构建初版
4. 人工调整细节

### 技巧 2：多生态对比分析

- 同时分析 2-3 个相关生态
- 例如："AI 大模型" vs "Crypto"
- 对比不动点、价值栈的异同

### 技巧 3：时间序列回溯

- 保存每个时间点的 JSON 文件
- 例如：2020 版、2022 版、2024 版
- 回看生态演化路径，验证预测准确性

## 总结

**记住**：
1. **原爆点 →** 定义生态起点
2. **不动点 →** 锁定投资锚点
3. **价值栈 →** 理解商业模式
4. **发展阶段 →** 判断当前主线
5. **连线关系 →** 把握价值流动

**这不是一次性作业，而是持续迭代的产品。**

开始你的第一个生态分析吧！
